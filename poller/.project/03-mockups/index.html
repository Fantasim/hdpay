Poller — Design Mockups

Crypto-to-Points microservice dashboard. These mockups define the visual design system and screen layouts for the SvelteKit dashboard. Component styles (tokens.css, components.css) are designed for reuse in the actual build.

Component Gallery

  • All Components Colors, typography, buttons, forms, tables, badges, stat cards, alerts, pagination, and more

Screens

  • Login Username + password authentication form with error state and network badge
  • Overview Dashboard Stats cards (8 metrics), time range selector, 7 chart placeholders (USD, points, chain/token breakdown, tiers)
  • Transactions Full history table with chain/token/status/tier filters, pagination, explorer links, and status badges
  • Watches Active and historical watches with status/chain filter chips, countdown timers, and poll results
  • Pending Points Accounts with unclaimed/pending points, summary cards, per-address breakdown with all-time totals
  • Errors Discrepancy detection (5 checks), stale pending transactions, and system error log with severity indicators
  • Settings Tier editor (9 tiers), IP allowlist management, watch defaults, and system info display

Design Decisions

Theme Dark (Linear-inspired, matches HDPay)
Primary Color #5e6ad2 (Blue-violet)
UI Font Inter
Data Font JetBrains Mono
Sidebar Full (240px, always visible)
Auth Session cookie (login page)
Responsive No (desktop-only admin tool)
Min Font Size 12px
